Утилита для поиска и замены подстрок

Янин Ярослав Иванович (Группа МК-101)

Командная утилита на языке C для поиска и замены последовательностей байт внутри файлов. Программа работает как с обычным текстом, так и с бинарными данными, обеспечивая безопасную перезапись без потери исходной информации.

Особенности:
* Потоковая обработка: Чтение входного файла осуществляется фиксированными блоками по N байт, что позволяет обрабатывать файлы любого размера без переполнения оперативной памяти.
* Сложные перекрытия: Конечный автомат корректно обрабатывает перекрывающиеся (вложенные) совпадения строк и частичные совпадения без потери "придержанных" байт.
* Поддержка спецсимволов: Искомая строка и строка для замены могут содержать шестнадцатеричные коды символов в формате \xNN (например, \x0A для переноса строки). Обратный слеш экранируется как \\.

Использование
Программа принимает 4 обязательных аргумента командной строки:
<input_file> <output_file> <search_seq> <replace_seq>

* <input_file> — путь к исходному файлу для чтения.
* <output_file> — путь к итоговому файлу для записи.
* <search_seq> — искомая последовательность символов или hex-кодов.
* <replace_seq> — последовательность для замены.

Сборка проекта
Для сборки проекта используется система CMake. Настройка проекта для сборки в Visual Studio (выполнять команды в консоли в корневой директории проекта):
> mkdir build
> cd build
> cmake -S .. -G "Visual Studio 17 2022" -A Win32

Примечание: Также можно просто открыть файл CMakeLists.txt напрямую в Visual Studio.

Сборка проекта через консоль:
> cmake --build . -C Release
> cmake --build . -C Debug

Запуск тестов
Логика конечного автомата и парсинг виртуальных строк покрыты юнит- и интеграционными тестами (10 проверок, фреймворк Unity). Команды для запуска тестирования:
> cmake --build . --target test_lib
> cmake --build . --target test_src
> cmake --build . --target test_all
> ctest -C Release
> ctest -C Debug
